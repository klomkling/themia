# Themia 0.2.0 — Tier 2: `Themia.Mediator` + mediator source-generator fold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended)
> or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`)
> syntax for tracking.

**Goal:** Port Zenity v1 `Zenity.Mediator` into `Themia.Mediator` (net10.0) — the request/response mediator
(abstractions, attributes, 5 pipeline behaviors, caching infrastructure, DI extension) — by relocation +
namespace/using rename, **and fold the v1 mediator handler-discovery generator into the existing
`Themia.SourceGenerator`** (netstandard2.0) as a SECOND `IIncrementalGenerator` pipeline alongside the DI
generator. The generated dispatcher is what makes `IMediator.SendAsync` work, so the generator fold is the
milestone's highest-risk item.

**Architecture:** Tier 2 of the framework-core layer (spec:
`docs/superpowers/specs/2026-06-02-themia-0.2.0-framework-rename-design.md`, §3b + §8a). `Themia.Mediator`
is **Zenity v1, canonical** — a near-verbatim port (namespace `Zenity.Mediator.* → Themia.Mediator.*`) with
its cross-cutting usings repointed onto the already-ported Tier-1 packages (`Themia.Caching`,
`Themia.Logging`) and the Tier-0 DI attribute package (`Themia.DependencyInjection`). It is **net10.0**
(framework TFM policy). The `[GenerateMediatorHandlers]` opt-in attribute and the `[SingletonHandler]` /
`[TransientHandler]` lifetime attributes live in the **`Themia.Mediator`** runtime package (NOT in the
generator) so the generator can match them by fully-qualified name on the consumer compilation. The generator
is referenced as an analyzer (`OutputItemType="Analyzer" ReferenceOutputAssembly="false"`); because the
generator must compile first and its tests need the `Themia.Mediator` abstractions to exist, the task order is
(a) abstractions+attributes → (b) generator fold+retarget+tests → (c) behaviors+infra+DI → (d) integration
tests that exercise the generated dispatcher.

**Audit results (resolve the spec's deferred Tier-2 decisions — recorded here, do not re-litigate):**

1. **`TransactionBehavior` → PORT NOW (not deferred to Tier 3).** Read of
   `Behaviors/TransactionBehavior.cs` confirms it is a **self-contained placeholder**: 31 lines,
   implements `IPipelineBehavior<TRequest,TResponse>`, and its `HandleAsync` body is
   `ArgumentNullException.ThrowIfNull(...)` then `return await next(cancellationToken)`. Its only `using`s
   are `Zenity.Mediator.Abstractions` + `Zenity.Mediator.Pipelines`. There is **no** `IUnitOfWork`,
   `DbContext`, or any data/transaction abstraction — nothing that waits on Tier 3
   (`Themia.Framework.Data.EFCore`). → Port it now with the other four behaviors.

2. **`TenantBehavior` / `ISkipTenantValidation` → DEFER (do not invent).** Neither exists in the v1
   `Zenity.Mediator` source (the full 30-file inventory is below; no `TenantBehavior.cs` /
   `ISkipTenantValidation.cs`). 0.2.0 ports what exists; it does not invent. Explicitly out of scope.

**Tech Stack:** .NET 10 (`net10.0`) for `Themia.Mediator`; netstandard2.0 (Roslyn, build-time) for the
`Themia.SourceGenerator` additions; `FluentValidation`, `Microsoft.Extensions.Logging`; project references
to `Themia.Caching`, `Themia.Logging`, `Themia.DependencyInjection`; xUnit; central package management;
`PublicAPI.*` surface tracking; **plain xUnit `Assert.*`** for all tests (NO Shouldly, NO NSubstitute — the
ported v1 tests are rewritten per transform rule 8); `GetDiagnostics()` error-free compile gate for the
generator tests (asserted via `Assert.Empty(result.Diagnostics)` over `RunGenerator` output — no Verify
snapshots); `Directory.Build.props` (Nullable, ImplicitUsings, TreatWarningsAsErrors,
GenerateDocumentationFile).

### Transform rules (apply to EVERY ported file)

1. **Namespace declarations:** `namespace Zenity.Mediator` → `namespace Themia.Mediator`
   (all sub-namespaces: `.Abstractions`, `.Behaviors`, `.Configuration`, `.Infrastructure`, `.Attributes`,
   `.Extensions`, `.Pipelines`). For the generator: `namespace Zenity.SourceGenerator.* → Themia.SourceGenerator.*`.
2. **Cross-reference usings (Mediator-internal):** `using Zenity.Mediator.*` → `using Themia.Mediator.*`.
3. **Cross-cutting using repoints (Mediator):**
   - `using Zenity.Core.Caching;` → `using Themia.Caching;` and type `IZenityCacheProvider → IThemiaCacheProvider`.
   - `using Zenity.Core.Caching.Extensions;` → `using Themia.Caching.Extensions;` and call `AddZenityCaching() → AddThemiaCaching()`.
   - `using Zenity.Core.Logging;` → `using Themia.Logging;` and type `ZenityLogContext → ThemiaLogContext`.
   - `Zenity.Primitives` DI attrs (`Singleton`/`Transient`/`Scoped`) → `Themia.DependencyInjection`
     (these surface only in the **generator's** `GetLifetime` FQN matching; no Mediator runtime file
     currently uses them — verify on port).
4. **Remove redundant explicit usings** covered by `ImplicitUsings` (`System`, `System.Collections.Generic`,
   `System.Linq`, `System.Threading`, `System.Threading.Tasks`) in the **net10.0** Mediator project. **Keep
   all explicit usings in the generator** — it is netstandard2.0 with `ImplicitUsings` off; the v1 generator
   files already carry `using System.Collections.Generic; using System.Collections.Immutable; using System.Linq;`
   etc. Leave them.
5. **Generator FQN/emit retargeting:** apply the §B retargeting-deltas table exactly. Match strings are
   compared verbatim against `OriginalDefinition.ToDisplayString()` / `SymbolDisplayFormat.FullyQualifiedFormat`,
   so a single character off silently disables discovery — copy the table values literally.
6. **No other edits.** Do not "improve" ported code. Port real framework types only (audit from clean source:
   `-not -path '*/obj/*' -not -path '*/bin/*'`).
7. **Test inputs:** rename `Zenity.Mediator.* → Themia.Mediator.*` in test `using`s and in-test type
   references; for the generator test, retarget the **expected output** strings the test asserts on
   (emitted namespaces, hint names, `AddGeneratedMediatorHandlers`, `MediatorDispatcher`) and the
   retargeted diagnostic IDs (`ZENMED00x → THEMIA…`, per the §B descriptor IDs chosen in Task 3 Step 6).
   Keep behavior identical otherwise.
8. **Rewrite ported tests to plain xUnit (Themia uses ONE test style — plain xUnit `Assert.*`; NO
   Shouldly, NO NSubstitute).** Every ported test that uses Shouldly or NSubstitute MUST be rewritten:
   Shouldly fluent assertions → `Assert.*`, and NSubstitute mocks → small hand-written fakes/spies.
   FluentValidation is **NOT** a test mock — it is a runtime dependency of `ValidationBehavior` (the v1
   tests subclass `AbstractValidator<T>` as real validators); it stays. Apply the two mapping tables below
   verbatim.

   **8a. Shouldly → xUnit `Assert.*` mapping** (covers every Shouldly method used across the ported
   tests — `ValidationBehaviorTests`, `CachingBehaviorTests`, `AttributeCacheMetadataProviderTests`
   (Caching + Infrastructure), `DefaultCacheKeyFactoryTests`, `MediatorIntegrationTests`,
   `ServiceRegistrationIntegrationTests`, `MediatorHandlerGeneratorTests`):

   | Shouldly (v1) | xUnit (Themia) |
   |---|---|
   | `actual.ShouldBe(expected)` | `Assert.Equal(expected, actual)` |
   | `actual.ShouldNotBe(expected)` | `Assert.NotEqual(expected, actual)` |
   | `actual.ShouldBe(expected, "msg")` | `Assert.Equal(expected, actual)` (drop the message; xUnit equality has no message overload) |
   | `actual.ShouldBeNull()` | `Assert.Null(actual)` |
   | `actual.ShouldNotBeNull()` | `Assert.NotNull(actual)` |
   | `actual.ShouldBeTrue()` / `.ShouldBeTrue("msg")` | `Assert.True(actual)` / `Assert.True(actual, "msg")` |
   | `actual.ShouldBeFalse()` / `.ShouldBeFalse("msg")` | `Assert.False(actual)` / `Assert.False(actual, "msg")` |
   | `value.ShouldBeGreaterThan(n)` | `Assert.True(value > n)` |
   | `collection.ShouldContain(item)` | `Assert.Contains(item, collection)` |
   | `str.ShouldContain(substr)` | `Assert.Contains(substr, str)` |
   | `str.ShouldStartWith(prefix)` | `Assert.StartsWith(prefix, str)` |
   | `sequence.ShouldContain(d => d.Id == "X", "msg")` (predicate overload, used in generator tests) | `Assert.Contains(sequence, d => d.Id == "X")` |
   | `collection.ShouldBeEmpty()` | `Assert.Empty(collection)` |
   | `actual.ShouldBeOfType<T>()` | `Assert.IsType<T>(actual)` |
   | `actual.ShouldBeSameAs(other)` / `.ShouldBeSameAs(other, "msg")` | `Assert.Same(other, actual)` |
   | `actual.ShouldNotBeSameAs(other)` / `.ShouldNotBeSameAs(other, "msg")` | `Assert.NotSame(other, actual)` |
   | `Should.Throw<T>(() => …)` | `Assert.Throws<T>(() => …)` |
   | `Should.ThrowAsync<T>(() => …)` (awaited) | `await Assert.ThrowsAsync<T>(() => …)` |

   Notes: where Shouldly assertions carried an explanatory string (`ShouldBeTrue("…")`, `ShouldBeSameAs(x, "…")`),
   keep it only on the `Assert.*` overloads that accept a `userMessage` (`Assert.True/False(cond, msg)`);
   for the others drop the string. The generator tests already mix in raw boolean predicates
   (`result.GeneratedTrees.Any(t => …)`) — those become `Assert.True(...)` / `Assert.False(...)`.

   **8b. NSubstitute → hand-written fake/spy.** NSubstitute appears in exactly ONE ported file —
   `CachingBehaviorTests.cs` — and ONLY to stub `ILogger<T>` (`Substitute.For<ILogger<CacheKeyIndex>>()`
   and `Substitute.For<ILogger<CachingBehavior<TRequest,TResponse>>>()`); no `.Returns(...)`, `.Received(n)`,
   or `.DidNotReceive()` is used anywhere (the logger is only passed in, never asserted on). The cache
   provider, key factory, metadata provider, key index, and distributed-lock provider are ALREADY real
   hand-written objects in the v1 tests (`InMemoryTestCacheProvider`, `DefaultCacheKeyFactory`,
   `AttributeCacheMetadataProvider`, `CacheKeyIndex`, `InMemoryDistributedLockProvider`) — NOT mocks — so
   they need no replacement. Replace each `Substitute.For<ILogger<T>>()` with the shared no-op test double
   `NullTestLogger<T>` (full code in Task 6 Step 1). Because no `.Received()`/`.DidNotReceive()` assertion
   exists, a no-op logger (not a counting spy) is sufficient and faithful to the original test intent.
   `InMemoryTestCacheProvider` is itself the cache-provider test double (a counting spy already exposing
   `GetCallCount`/`SetCallCount`/`RemoveCallCount`) and is ported verbatim — it covers every cache
   assertion the tests make.

**Explicitly NOT ported:** `TenantBehavior` / `ISkipTenantValidation` (do not exist in v1 — see audit);
v1 generator's `Discovery/ServiceDiscovery.cs`, `Models/ServiceModel.cs`, and DI-emit portions of
`SourceBuilder.cs` (already represented by Themia's `ServiceRegistrationGenerator`).
