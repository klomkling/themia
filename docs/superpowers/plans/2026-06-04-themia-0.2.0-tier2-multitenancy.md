# Themia 0.2.0 — Tier 2: `Themia.MultiTenancy` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended)
> or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`)
> syntax for tracking.

**Goal:** Port zenity-v1 `Zenity.MultiTenancy` into `Themia.MultiTenancy` (net10.0) — the tenant
**resolution stack** (strategies, stores, resolver, accessor, builder/DI, middleware) — **rewired
onto** v2's canonical `Themia.Framework.Core` tenancy. This is the milestone's **headline
reconciliation** (spec §3a / §8a): v1's rich `TenantInfo`/`ITenantAccessor` are **kept** (v2's
`TenantContext` is *not* a superset of them — see below), and a single ported
`TenantResolutionMiddleware` **bridges** resolution into Framework.Core's static
`TenantContextAccessor` so the Tier-3 EF global query filter keys off the same tenant.

**Architecture:** Tier 2 of the framework layer (spec:
`docs/superpowers/specs/2026-06-02-themia-0.2.0-framework-rename-design.md`, §3a + §8a).
`Themia.MultiTenancy` is **net10.0** (framework TFM policy), references `Themia.Framework.Core`
(Tier 1, for `TenantId` + `TenantContextAccessor`), and pulls `Microsoft.AspNetCore.Http.Abstractions`
(middleware + `IApplicationBuilder`), `Microsoft.Extensions.Caching.Memory` (cached store),
`Microsoft.Extensions.Options`/`.Logging`/`.DependencyInjection` (already pinned), and `Dapper`
(sanctioned tenant-catalog store). It must **not** depend on EF Core — Tier 3 consumes the
connection-string seam, Tier 2 only carries it.

**The reconciliation (spec §8a — decided 2026-06-04; do NOT relitigate):**

- **KEEP v1 `TenantInfo` / `ITenantAccessor`** and the entire resolution stack. The audit found v2's
  `TenantContext` holds only a validated `TenantId` + `Source`, whereas v1's `TenantInfo` carries
  `Name` / `Environment` / **`ConnectionString`** / `Properties` plus resolution traceability
  (`TenantResolutionResult`). v2 is **not** a superset, so per §3a's own conditional v1's model is kept
  and lives in `Themia.MultiTenancy`.
- **THE BRIDGE.** On successful resolution the single ported `TenantResolutionMiddleware` sets **both**:
  - `ITenantAccessor.Current = <resolved TenantInfo>` — v1 behavior; the rich model carrying the
    per-tenant `ConnectionString`; **and**
  - `Themia.Framework.Core.Abstractions.Tenancy.TenantContextAccessor.CurrentTenantId =
    TenantId.From(tenantInfo.Identifier)` — the **[INTRODUCED]** bridge line so the Tier-3 EF global
    query filter (which reads the static `TenantContextAccessor`) keys off the same tenant. This is the
    one delta over the verbatim v1 middleware and ships with a dedicated test (Task 9).
  - **DROP** v1's `_ = context.RequestServices.GetService<TenantContext<string, string>>();` line that
    hydrated `Zenity.Core.Context.TenantContext<,>` — that internal generic type is **not** ported.

**Tier-3 contract (pin this for the next tier):** `ITenantAccessor.Current.ConnectionString` is the
**load-bearing seam** for DB-per-tenant. The first Themia consumer (EzyAssets) offers agency customers
a **separate database per company**; the Tier-3 `Themia.Framework.Data.EFCore` `DbContext` factory
**reads `ITenantAccessor.Current.ConnectionString`** to construct the per-tenant context. Tier 2
*resolves and carries* the connection string + sets the bridge `TenantId`; Tier 3 *consumes* both. The
boundary holds — `Themia.MultiTenancy` adds **no** EF Core dependency.

**Tech Stack:** .NET 10 (`net10.0`); `Themia.Framework.Core` (ProjectReference);
`Microsoft.AspNetCore.Http.Abstractions`; `Microsoft.Extensions.{Caching.Memory,Options,Logging,
DependencyInjection,DependencyInjection.Abstractions}`; `Dapper`; **plain xUnit only** (no Shouldly, no
NSubstitute) + `Microsoft.AspNetCore.TestHost` + `Microsoft.Data.Sqlite` (tests); central package management;
`PublicAPI.*` surface tracking; `Directory.Build.props` (Nullable, ImplicitUsings,
TreatWarningsAsErrors, GenerateDocumentationFile).

### Transform rules (apply to EVERY ported file)

1. **Namespace declarations & sub-namespaces:** `namespace Zenity.MultiTenancy(.X) →
   Themia.MultiTenancy(.X)` (`.Abstractions`, `.AspNet`, `.Internal`, `.Options`*, `.Stores`,
   `.Strategies`). *(`MultiTenancyOptions(.Validator)` declare `namespace Zenity.MultiTenancy` directly
   — they become `Themia.MultiTenancy`.)*
2. **Cross-reference usings:** `using Zenity.MultiTenancy.* → using Themia.MultiTenancy.*`.
3. **Repoint v1 tenant-CONTEXT usings:** drop `using Zenity.Core.Context;` everywhere it appears
   (`MultiTenancyBuilder.cs`, `AspNet/TenantResolutionMiddleware.cs`). Where the bridge is needed, add
   `using Themia.Framework.Core.Abstractions.Tenancy;` (`TenantId`, `TenantContextAccessor`). v1's
   `ITenantAccessor`/`TenantInfo` are **kept** and stay in `Themia.MultiTenancy.Abstractions` — only the
   middleware bridge writes into Framework.Core's static accessor.
4. **No `Zenity.Primitives` / `Zenity.Core.Caching` usings exist** in this package (audited clean) — no
   repoint needed for those. (Rule retained for parity with the spec's transform list.)
5. **Remove redundant explicit usings** covered by `ImplicitUsings` (`System`,
   `System.Collections.Generic`, `System.Linq`, `System.Threading`, `System.Threading.Tasks`). **Keep**
   non-implicit usings: `System.Collections.Concurrent`, `System.Collections.ObjectModel`, `System.Data`,
   `System.Text.RegularExpressions`, `Microsoft.AspNetCore.Builder`, `Microsoft.AspNetCore.Http`,
   `Microsoft.Extensions.Caching.Memory`, `Microsoft.Extensions.Options`,
   `Microsoft.Extensions.Logging`, `Microsoft.Extensions.DependencyInjection(.Extensions)`, `Dapper`,
   and (new) `System.Text.Json.Serialization` in `TenantModels.cs`.
6. **Type renames:** `AddZenityMultiTenancy → AddThemiaMultiTenancy`,
   `UseZenityMultiTenancy → UseThemiaMultiTenancy` (declarations + all test call sites). No other type
   renames.
7. **Drop demo/benchmark fixtures.** Port real types only; audit from clean source
   (`-not -path '*/obj/*' -not -path '*/bin/*'`). The only non-shippable items dropped are the two
   `Zenity.Core.Context`-coupled members noted above.
8. **Test inputs:** apply rules 1–2, 6 to test `using`s/call sites; keep the *behavior/intent* of each
   assertion identical, **but rewrite the assertion/mocking style to plain xUnit per transform rule 9**
   (the Themia repo is xUnit-only — no Shouldly, no NSubstitute). Drop `using Shouldly;` and
   `using NSubstitute;` from every ported test file.

9. **Rewrite ported tests to plain xUnit** (repo decision — Themia ships ONE test style: plain xUnit
   `Assert.*` + hand-written test doubles; matches the Tier-1 `Themia.Caching.Tests` style). Apply to
   every ported test file. Do **not** change what each test asserts — only how.

   **9a. Shouldly → xUnit `Assert.*`** (every Shouldly method actually used in these source files, plus
   the two `ShouldNotContain` forms the INTRODUCED redaction tests use):

   | Shouldly (v1) | Plain xUnit |
   |---|---|
   | `x.ShouldBe(y)` | `Assert.Equal(y, x)` |
   | `x.ShouldNotBe(y)` | `Assert.NotEqual(y, x)` |
   | `x.ShouldBeNull()` | `Assert.Null(x)` |
   | `x.ShouldNotBeNull()` | `Assert.NotNull(x)` |
   | `x.ShouldBeTrue()` | `Assert.True(x)` |
   | `x.ShouldBeFalse()` | `Assert.False(x)` |
   | `x.ShouldContain(y)` (string) | `Assert.Contains(y, x)` |
   | `x.ShouldContain(y)` (collection) | `Assert.Contains(y, x)` |
   | `x.ShouldNotContain(y)` (string) | `Assert.DoesNotContain(y, x)` |
   | `x.ShouldNotContain(y, Case.Insensitive)` | `Assert.DoesNotContain(y, x, StringComparison.OrdinalIgnoreCase)` |
   | `x.ShouldBeEmpty()` | `Assert.Empty(x)` |
   | `x.ShouldBeOfType<T>()` | `Assert.IsType<T>(x)` (returns the cast value when needed) |
   | `x.ShouldBeGreaterThan(y)` | `Assert.True(x > y)` |
   | `x.ShouldBeLessThanOrEqualTo(y)` | `Assert.True(x <= y)` |
   | `collection.ShouldAllBe(p => …)` | `Assert.All(collection, item => Assert.True(p(item)))` |
   | `Should.Throw<T>(() => …)` | `Assert.Throws<T>(() => …)` |
   | `Should.ThrowAsync<T>(async () => …)` | `await Assert.ThrowsAsync<T>(async () => …)` |

   Notes: Shouldly reads subject-first (`actual.ShouldBe(expected)`); xUnit `Assert.Equal` is
   **expected-first** (`Assert.Equal(expected, actual)`) — swap the operands. `Case.Insensitive` is a
   `Shouldly` enum; drop the `using Shouldly;` and use `StringComparison.OrdinalIgnoreCase` instead (the
   no-top-N guard in Task 6 uses this).

   **9b. NSubstitute → real framework no-op types / hand-written fakes.** The audit shows NSubstitute is
   used **only for stubbing** (no `.Received()`/`.DidNotReceive()` verification anywhere), and only on
   two framework abstractions — both have a real, allocation-free replacement, so **no new hand-written
   fake class is needed**:

   | NSubstitute (v1) | Plain replacement |
   |---|---|
   | `Substitute.For<IOptions<MultiTenancyOptions>>()` + `opts.Value.Returns(new MultiTenancyOptions { … })` | `Microsoft.Extensions.Options.Options.Create(new MultiTenancyOptions { … })` (real `IOptions<T>` wrapper; `using Microsoft.Extensions.Options;`) |
   | `Substitute.For<ILogger<HeaderTenantResolutionStrategy>>()` | `NullLogger<HeaderTenantResolutionStrategy>.Instance` (`using Microsoft.Extensions.Logging.Abstractions;`) |

   `ITenantStore` is **not** NSubstitute-mocked in v1 — it is already covered by the hand-written
   `TestUtilities/FakeTenantStore.cs` (a real fake tracking `FindCallCount`); reuse the ported
   `FakeTenantStore` wherever a store double is needed (e.g. `CachedTenantStoreTests`,
   `DefaultTenantResolver` coverage). No `ITenantResolver` / `ITenantResolutionStrategy` /
   `ITenantAccessor` mocks exist in the v1 suite, so none need a fake.

   **Affected files** (those that used Shouldly and/or NSubstitute — all get the 9a rewrite; the four
   marked also get 9b): `Models/{TenantInfoTests,TenantResolutionContextTests,TenantResolutionResultTests}.cs`,
   `Configuration/{MultiTenancyOptionsTests,ServiceCollectionExtensionsTests}.cs`,
   `Strategies/{Header,Path,Default}TenantResolutionStrategyTests.cs` **(9b: `IOptions`/`ILogger`)**,
   `Stores/{InMemory,Cached,Dapper}TenantStoreTests.cs`, `AspNet/{TenantResolutionMiddlewareTests,MiddlewareIntegrationTests}.cs`.
   `TestUtilities/{FakeTenantStore,SqliteTestDb}.cs` use neither Shouldly nor NSubstitute — port verbatim


```csharp
using Microsoft.AspNetCore.Builder;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.Logging;
using Themia.Framework.Core.Abstractions.Tenancy;
using Themia.MultiTenancy.Abstractions;

namespace Themia.MultiTenancy.AspNet;

/// <summary>
/// Middleware that resolves the tenant from the incoming HTTP request, stores it in
/// <see cref="ITenantAccessor"/> (the rich model carrying the per-tenant connection string), and
/// bridges the resolved identifier into <see cref="TenantContextAccessor"/> so the data layer's
/// tenant query filter keys off the same tenant.
/// </summary>
public sealed class TenantResolutionMiddleware
{
    private readonly RequestDelegate _next;

    public TenantResolutionMiddleware(RequestDelegate next) =>
        _next = next ?? throw new ArgumentNullException(nameof(next));

    public async Task InvokeAsync(HttpContext context)
    {
        if (context is null)
        {
            throw new ArgumentNullException(nameof(context));
        }

        var resolver = context.RequestServices.GetRequiredService<ITenantResolver>();
        var accessor = context.RequestServices.GetRequiredService<ITenantAccessor>();
        var logger = context.RequestServices.GetRequiredService<ILogger<TenantResolutionMiddleware>>();

        var resolution = await resolver.ResolveAsync(new TenantResolutionContext(
            context.Request.Host.Host,
            context.Request.Path.Value ?? string.Empty,
            context.Request.Headers.ToDictionary(h => h.Key, h => h.Value.ToString(), StringComparer.OrdinalIgnoreCase),
            context.User.Claims.GroupBy(c => c.Type).ToDictionary(g => g.Key, g => g.First().Value)), context.RequestAborted);

        if (resolution is null)
        {
            logger.LogWarning("Tenant resolution returned null. Ensure strategies are configured.");
        }

        accessor.Current = resolution;

        // [INTRODUCED] Bridge the resolved tenant into the framework's ambient tenant context so the
        // Tier-3 EF Core global query filter keys off the same tenant. Dropped: v1's
        // TenantContext<string, string> hydration (that internal generic type is not ported).
        TenantContextAccessor.CurrentTenantId = TenantId.From(resolution?.Identifier);

        await _next(context);
    }
}

public static class TenantResolutionMiddlewareExtensions
{
    public static IApplicationBuilder UseThemiaMultiTenancy(this IApplicationBuilder app)
    {
        if (app is null)
        {
            throw new ArgumentNullException(nameof(app));
        }

        return app.UseMiddleware<TenantResolutionMiddleware>();
    }
```
