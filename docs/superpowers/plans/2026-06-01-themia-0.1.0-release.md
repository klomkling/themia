# Themia 0.1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to work this plan
> task-by-task. Steps use checkbox (`- [ ]`) syntax. **Several tasks are `[MAINTAINER]`** —
> nuget.org / GitHub-settings actions the agent cannot perform; do those manually and check them off.

**Goal:** Publish the first Themia release — `0.1.0` = `Themia.AspNetCore` — to nuget.org via
**Trusted Publishing (GitHub OIDC)**, with no long-lived `NUGET_API_KEY` secret.

**Architecture:** `.github/workflows/release.yml` auto-triggers on push to `main`; it reads
`<Version>` from `Directory.Build.props` (already `0.1.0`), and releases when no `v{version}` tag
exists. The release job runs in the `nuget` environment, mints a short-lived key via `NuGet/login@v1`
(OIDC), `dotnet pack`s the whole solution, `dotnet nuget push`es, then tags `v0.1.0` and creates the
GitHub Release. A **required-reviewer rule on the `nuget` environment** is the manual gate so the
auto-on-push trigger never publishes unintentionally.

**Tech stack:** GitHub Actions, NuGet Trusted Publishing (`NuGet/login@v1`), `dotnet pack`/`push`.

**Reference:** release strategy spec `docs/superpowers/specs/2026-06-01-themia-release-strategy-design.md`.

## Notes for the implementer

- **No `NUGET_API_KEY` secret exists or is needed** — Trusted Publishing mints a ~1h key via OIDC.
  If publish fails with "returned no key", the cause is a missing/mismatched Trusted Publishing policy
  (Task 3) or an unset `NUGET_USER` variable (Task 4), not a secret.
- **Recovery:** if a run tags `v0.1.0` but fails before publish (or vice-versa), re-run via
  `workflow_dispatch` with input `version: 0.1.0` — the workflow checks out the existing tag and
  re-publishes idempotently (`--skip-duplicate`, release `--clobber`).
- **Ordering matters:** do Tasks 3–5 *before* merging the Task 6 release commit, so the publish
  succeeds on the first approved run. Merging this plan's PR (Task 1) before the policy exists will
  produce one failed Release run — harmless.
